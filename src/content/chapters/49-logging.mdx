Logging ช่วยให้เราเห็นว่า API ทำงานอย่างไร เกิด error ที่ไหน และ request สำคัญเกิดขึ้นเมื่อไหร่

ใน ASP.NET Core เราจะใช้ `ILogger<T>` ซึ่งรองรับ structured logging และทำงานร่วมกับ logging providers ที่ framework มีให้ เช่น Console และ Debug

## วิธีเรียนบทนี้

บทนี้จะไม่สร้าง library logging ใหม่ แต่จะใช้ logging ที่ ASP.NET Core มีให้แล้ว:

1. เข้าใจ structured logging
2. inject `ILogger<AuthService>`
3. เพิ่ม log ใน register/login
4. inject `ILogger<AdminUserService>`
5. เพิ่ม log ใน admin action
6. ตั้งค่า `LogLevel`
7. รัน API แล้วอ่าน log จาก console

## สิ่งที่จะใช้ในบทนี้

| สิ่งที่จะใช้ | ความหมาย |
| --- | --- |
| `ILogger<T>` | logging service ที่ DI สร้างให้ตาม class ที่ใช้งาน |
| `LogInformation` | log เหตุการณ์ปกติที่สำคัญ |
| `LogWarning` | log เหตุการณ์ผิดปกติแต่ระบบยังทำงานต่อได้ |
| structured logging | log ที่มี placeholder เช่น `{UserId}` |
| `LogLevel` | ระดับความละเอียดของ log |

## หลังจบบทนี้ ไฟล์ที่เปลี่ยน

```text
Services/AuthService.cs
Services/AdminUserService.cs
appsettings.json
```

## ขั้นที่ 1: เข้าใจ structured logging

Structured logging คือการเขียน log โดยใช้ placeholder แทนการต่อ string เอง

ควรเขียนแบบนี้:

```csharp
logger.LogInformation(
    "User {UserId} logged in",
    user.Id);
```

ไม่ควรเขียนแบบนี้:

```csharp
logger.LogInformation("User " + user.Id + " logged in");
```

แบบแรกทำให้ log provider เข้าใจว่า `UserId` เป็น field หนึ่ง สามารถค้นหา filter หรือ query ต่อได้ง่ายกว่า

## ขั้นที่ 2: Inject logger เข้า AuthService

เปิด `Services/AuthService.cs`

แก้ constructor ให้รับ `ILogger<AuthService>`:

```csharp
public class AuthService(
    IUserRepository userRepository,
    IPasswordHasher<User> passwordHasher,
    JwtTokenService jwtTokenService,
    ILogger<AuthService> logger)
```

`ILogger<AuthService>` บอก framework ว่า log นี้มาจาก `AuthService` เวลาอ่าน log จะเห็น category ชัดขึ้น

## ขั้นที่ 3: เพิ่ม log ตอน register

ใน `RegisterAsync` เพิ่ม log ตอนเริ่ม request:

```csharp
logger.LogInformation(
    "Register requested for email {Email}",
    request.Email);
```

หลังสร้าง user สำเร็จ เพิ่ม log:

```csharp
logger.LogInformation(
    "User {UserId} registered with role {Role}",
    createdUser.Id,
    createdUser.Role);
```

อย่า log password หรือ password hash แม้จะเป็นช่วง development

## ขั้นที่ 4: เพิ่ม log ตอน login

ตอน login ไม่สำเร็จ เพิ่ม warning:

```csharp
logger.LogWarning(
    "Login failed for email {Email}: invalid credentials",
    request.Email);
```

ตอน login สำเร็จ เพิ่ม information:

```csharp
logger.LogInformation(
    "User {UserId} logged in",
    user.Id);
```

ใช้ `Warning` กับ login failed เพราะเป็นเหตุการณ์ที่ควรสังเกต แต่ยังไม่ใช่ error ของระบบเสมอไป

## ขั้นที่ 5: Inject logger เข้า AdminUserService

เปิด `Services/AdminUserService.cs`

แก้ constructor ให้รับ `ILogger<AdminUserService>`:

```csharp
public class AdminUserService(
    IUserRepository userRepository,
    CurrentUserService currentUserService,
    AuditLogService auditLogService,
    ILogger<AdminUserService> logger)
```

ถ้า constructor ของคุณมี dependency อื่นอยู่แล้ว ให้เพิ่ม `logger` ต่อท้ายตามรูปแบบเดียวกัน

## ขั้นที่ 6: เพิ่ม log ตอน admin เปลี่ยน role/status

หลังเปลี่ยน role สำเร็จ เพิ่ม log:

```csharp
logger.LogInformation(
    "Admin {AdminUserId} changed user {TargetUserId} role from {OldRole} to {NewRole}",
    currentAdminId,
    user.Id,
    oldRole,
    user.Role);
```

หลังเปลี่ยนสถานะสำเร็จ เพิ่ม log:

```csharp
logger.LogInformation(
    "Admin {AdminUserId} changed user {TargetUserId} active status from {OldStatus} to {NewStatus}",
    currentAdminId,
    user.Id,
    oldIsActive,
    user.IsActive);
```

Audit log เก็บข้อมูลลง database ส่วน application log ช่วยให้ทีมดูเหตุการณ์จาก console, file หรือ logging backend ได้

## ขั้นที่ 7: ตั้งค่า LogLevel

เปิด `appsettings.json`

ตรวจว่ามี configuration นี้:

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning",
      "Microsoft.EntityFrameworkCore.Database.Command": "Warning"
    }
  }
}
```

ความหมาย:

- `Default` คือระดับ log ของ application โดยรวม
- `Microsoft.AspNetCore` คุม log จาก framework
- `Microsoft.EntityFrameworkCore.Database.Command` คุม log SQL command จาก EF Core

ถ้าเปิด EF Core SQL command เป็น `Information` จะเห็น SQL เยอะมาก เหมาะกับ development มากกว่า production

## ขั้นที่ 8: รันและตรวจ log

รันจากโฟลเดอร์ `Backend.Api`

```powershell
dotnet run
```

จากนั้นยิง request เช่น register หรือ login แล้วดู output ใน terminal

สิ่งที่ควรเห็น:

- log register/login สำเร็จ
- log login failed เป็น `Warning`
- ไม่มี password, token, signing key หรือ connection string ใน log

## สิ่งที่ไม่ควร log

- password
- password hash
- JWT access token
- refresh token
- signing key
- connection string ที่มี password
- personal data ที่ไม่จำเป็นต่อการ debug

ถ้าต้อง log email เพื่อ debug ให้เข้าใจว่า email เป็นข้อมูลส่วนบุคคล ควรใช้เท่าที่จำเป็นและตรวจนโยบายของระบบจริง

## Checkpoint

ก่อนอ่านบทต่อไป ให้ตรวจว่าทำได้ครบตามนี้

- service สำคัญใช้ `ILogger<T>`
- log ใช้ placeholder เช่น `{UserId}` ไม่ต่อ string เอง
- login/register/admin action มี log ที่ช่วยตรวจสอบได้
- ไม่ log password, token หรือ secret
- `appsettings.json` มี `Logging:LogLevel`
